Construct a DepotCycle Device Lifecycle Marketplace supporting end-to-end device intake, internal movement, and resale operations for organizations that manage multiple campuses and a local fulfillment team. The Leptos web UI provides role-based workspaces: Shoppers browse and buy, Inventory Clerks create receiving/issuing/transfer/return/loan/scrap documents, Managers approve exceptions, Support Agents handle after-sales, and Administrators manage operations. Users can search listings with multi-criteria filters including category, price range in USD, condition, campus, and post time, then sort by relevance, popularity, approximate distance (calculated from locally stored ZIP-code coordinates), or price. The search bar offers type-ahead suggestions, recent-history recall, and a one-click “clear history” action. Personalized recommendations appear on the home and detail views based on locally recorded views, favorites, and orders; each recommendation includes a short “why you’re seeing this” explanation and can be disabled per user. Listings and help content are organized through a multi-level taxonomy with tags, keywords, topic aggregation pages, and editable SEO fields so staff can control title/description/slugs for internal discovery. Media supports large video/photo uploads with chunking and resume; users see progress, retry, and checksum validation feedback, and playback uses authenticated, time-limited links to prevent reuse outside the app. Logistics screens let staff create shipment orders, assign carriers, enter tracking numbers, confirm ship/receive, and drive after-sales flows for return/exchange/refund with evidence uploads, clear status timelines, and SLA timers (for example, first response due within 1 business day and final decision within 3 business days).

The backend uses Axum to expose offline REST-style APIs consumed by the Leptos frontend, and SQLite persists all data locally so the system runs fully without internet access (single machine or LAN-hosted server). Authentication is strictly username/password with Argon2-hashed passwords, minimum 12 characters, and a 15-minute lockout after 5 failed attempts; sessions expire after 30 minutes of inactivity. Document processing enforces strong linkage between each document and device line items: every receiving/issuing/transfer/return/loan/scrap action updates on-hand quantities and device state, and automatically writes immutable ledger change records with before/after values, operator, timestamp, and related document reference; managers must approve any document that changes inventory value by more than $2,500.00 or scraps more than 5 units at once. Logistics and after-sales use explicit state transitions (created, packed, shipped, received, completed, canceled; and requested, evidence_pending, under_review, approved, rejected, closed) with validation to prevent skipping steps; tracking API “integration points” are defined but remain disabled by default, requiring manual updates to stay offline-compliant. The admin console includes local credential and companion credential management, ratings review with appeal tickets, template/content configuration, and announcements delivered as in-app notifications (no external push). Dashboards compute retention/activity/conversion/service-quality from local event logs, while A/B tests and canary releases are implemented via local feature flags and cohort assignment stored in SQLite. Uploaded media is stored on local disk with MIME/type checks and SHA-256 fingerprints; sensitive fields (names, phone numbers) are masked in UI logs and encrypted at rest, with audit trails for all admin actions.
